Testing

To test Ditto using kotlin, we provide you with some basic convenience functions you can use to get started.

For code that you can use in your project, see the example on GitHub.

Create custom directories​

Each instance of Ditto should use a different directoriy. You can extend DefaultAndroidDittoDependencies to provide a custom directory.

/** * Implementation of `AndroidDittoDependencies` which uses a custom `persistenceDirectory`. */data class CustomDirectoryAndroidDittoDependencies constructor(    private val androidDittoDependencies: DefaultAndroidDittoDependencies,    private val customDir: File): AndroidDittoDependencies {
    override fun persistenceDirectory(): String {        return customDir.path    }
    override fun ensureDirectoryExists(path: String) {        androidDittoDependencies.ensureDirectoryExists(path)    }
    override fun context(): Context {        return androidDittoDependencies.context()    }
}

Provide the context from the InstrumentationRegistry​

Ditto needs the current context, which can be retrieved as such:


import androidx.test.platform.app.InstrumentationRegistryimport java.io.File
/**  * Creates a set of Android dependencies combining defaults with a custom directory.  */fun dependenciesWithCustomDirectory(customDir: File): DittoDependencies {    val context: Context = InstrumentationRegistry.getInstrumentation().targetContext    val dependencies = DefaultAndroidDittoDependencies(context)    return CustomDirectoryAndroidDittoDependencies(dependencies, customDir)}

Initialize ditto​

You can create a default identity -- we use onlinePlayground here, but you should test with whatever identity you are using in production.

@JvmOverloadsfun getDitto(    dependencies: DittoDependencies,    identity: DittoIdentity = DittoIdentity.OnlinePlayground(            appID = "REPLACE_ME",             dependencies = androidDependencies,             token = "REPLACE_ME"    ),    offlineLicense: String? = testLicense,): Ditto {    val ditto = Ditto(dependencies, identity)    return ditto}

Create your test file​

  1. Initialize two ditto instances with different directories.
  2. Call startSync() on both instances.
package live.ditto.dittotesting
import org.junit.Assert.assertEqualsimport org.junit.Testimport java.io.File

class ExampleDittoTest () {    @Test    fun twoDittos() {        val ditto1 = getDitto(dependenciesWithCustomDirectory(File(getWorkDir(), "ditto1")))        val ditto2 = getDitto(dependenciesWithCustomDirectory(File(getWorkDir(), "ditto2")))
        ditto1.startSync()        ditto2.startSync()    }}

Listen for changes to data​

Using subscribe(), we listen to changes to data. We write tests as a response to the observeLocal() function based on what we expect to happen.

The tests should reside within the callback so that they properly test the state of the database after synchronization has completed. 

...val coll1 = ditto1.store.collection("cars")val coll2 = ditto2.store.collection("cars")val docId = coll1.upsert(mapOf(    "make" to "toyota",    "mileage" to 160000))coll2.findById(docId).subscribe()coll2.findById(docId).observeLocal { doc, event ->    if (!event.isInitial) {        assertEquals(doc["make"], "toyota")    }}

Clean up​

Don't forget to clean up between tests. This can also be implemented as part of a base test class.

@JvmOverloadsfun clearUpDittoDirectories(context: Context, customDir: String) {    val dittoDir = File(customDir)    dittoDir.deleteRecursively()}

@Testfun twoDittos() {    val ditto1 = getDitto(dependenciesWithCustomDirectory(File(getWorkDir(), "ditto1")))    val ditto2 = getDitto(dependenciesWithCustomDirectory(File(getWorkDir(), "ditto2")))
    ditto1.startSync()    ditto2.startSync()
    val coll1 = ditto1.store.collection("cars")    val coll2 = ditto2.store.collection("cars")    val docId = coll1.upsert(mapOf(        "make" to "toyota",        "mileage" to 160000    ))    val liveQuery = coll2.findByID(docId).subscribe()    coll2.findByID(docId).observeLocal { doc, event ->        if (!event.isInitial) {            assertEquals(doc!!["make"], "toyota")            ditto1.stopSync()            ditto2.stopSync()        }    }}
